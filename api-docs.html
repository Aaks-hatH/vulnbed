VulnBank Accounts API

REST API Documentation

Base URL: https://vulnbank.example.com/api/v1 — No authentication required on most endpoints

Authentication
POST /api/v1/auth/login No Auth
Authenticate user. Returns JWT token. No rate limiting. 
{ "username": "admin@vulnbank.com", "password": "password123" }
Response: 
{ "token": "eyJhbGci...", "user_id": 1, "role": "admin" }
GET /api/v1/auth/reset?email=user@example.com&redirect=http://evil.com Open Redirect
Password reset. The redirect parameter is not validated — open redirect vulnerability. Attacker can use: /api/v1/auth/reset?email=victim@bank.com&redirect=https://phishing.example.com
Users
GET /api/v1/users No Auth — IDOR
Returns all users. No authentication required. 
[{"id":1,"name":"Admin","email":"admin@vulnbank.com","balance":98450},
 {"id":2,"name":"Alice","email":"alice@example.com","balance":12300}]
GET /api/v1/users/{id} IDOR
Get user by ID. No ownership check — any ID works. Try: api/v1/users/1, api/v1/users/2
GET /api/v1/users/{id}/transactions IDOR
Returns all transactions for any user ID. No authentication.
Transfers
POST /api/v1/transfer CSRF Vulnerable
Execute money transfer. No CSRF token, no SameSite cookie. 
{ "from_account": "ACC001", "to_account": "12345678", "amount": 100.00 }
GET /api/v1/accounts?user_id=1 IDOR
Get account list for any user. Change user_id to access other users' accounts.
Admin
GET /api/v1/admin/users No Auth Required
Returns all users including password hashes and PII. No authentication check.
DELETE /api/v1/admin/users/{id} No Auth
Delete any user account. No authentication, no CSRF protection.
